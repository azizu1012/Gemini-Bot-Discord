================================================================================
2) RUNTIME OVERVIEW
================================================================================

Entry points:
- main.py (thin entrypoint)
- run_bot.py (launcher chính, bot-only hoặc bot + server)
- run_bot.sh (launcher Linux/Ubuntu + PM2 modes: --pm2, --pm2-fresh)

Luồng xử lý chính nằm ở:
- src/handlers/message_handler.py (routing, context, intent detection)
- src/core/gemini_api_manager.py (API key pool, throttle, client pool, error classification)
- src/core/gemini_pipeline.py (reasoning loop, final synthesis, fallback)

Flow tóm tắt hiện tại:
1. Nhận message (DM hoặc mention).
2. Build context/history từ DB.
3. Tính signal đa user từ DB (distinct user count + has other users).
4. Nếu admin + query cross-user detail: build evidence block từ DB.
5. Nếu có file đính kèm: chạy file indexing pipeline (chunk → LLM index → Flash validation).
6. Chạy reasoning loop (có thể gọi tools) qua GeminiPipeline + GeminiApiManager.
7. Chạy final synthesis (và fallback) với cùng privacy context.
8. Persist user/assistant message vào DB.

5) FILE INDEXING PIPELINE
================================================================================

Module chính: src/services/file_index_service.py

Flow:
1. Upload file → download + chunk (via file_parser.py).
2. Mỗi chunk → LLM reasoning model index (JSON format, map theo filename thực).
3. Index được validate bằng Flash model trước khi cho chat tiếp.
4. Index lưu JSON tại FILE_INDEX_DIR.

Kỹ thuật:
- INDEX_MAX_OUTPUT_TOKENS = 65000 (5× reasoning budget cho heavy files).
- Dependency injection: FileIndexService nhận API callables từ GeminiApiManager.
- Helper functions nằm ngoài class (build_index_base_name, write_index_json, read_index_json,
  should_use_last_index, build_index_context).
- Prompt templates nạp từ src/instructions/file_index_*.txt.

================================================================================
9) WHEN TO STAY SQLITE VS MOVE POSTGRES
================================================================================

SQLite là đủ khi:
- single-node deployment,
- traffic vừa,
- ưu tiên đơn giản vận hành.

Nên cân nhắc PostgreSQL khi:
- concurrency tăng rõ,
- cần scale nhiều process/instance,
- cần observability/transaction control cao hơn.

Khuyến nghị migration:
- Giữ interface repository ổn định.
- Chuẩn bị adapter để thay backend DB dần.

## Custom API Wrapper Capabilities
- Perfect Emulation of Google GenAI SDK: The OpenAI wrapper correctly formats `CustomAPIWrapperResponse` so that multiple parallel `tool_calls` are mapped into a native list of `candidates[0].content.parts`, ensuring tools work without any modification.
- Parallel Tool Execution: The `gemini_pipeline.py` architecture has been upgraded to scan through all response parts simultaneously and execute all tools in parallel or sequentially within a single turn, before returning the batched `function_response` block to the model.
- Health Check Monitoring: Supports fully automated background pinging for custom APIs via `HealthCheckerService`, with an LLM generating analysis summaries when an API endpoint changes state (alive/dead).
