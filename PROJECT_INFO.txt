================================================================================
1) RUNTIME OVERVIEW
================================================================================

Entry points:
- `main.py`
- `run_bot.py`
- `run_bot.sh`

Core flow trong `src/handlers/message_handler.py`:
1. Nhận message (DM hoặc mention).
2. Kiểm tra quyền/giới hạn/tải file đính kèm nếu có.
3. Gọi pipeline Gemini 2 tầng:
   - Tier 1: `_call_gemini_reasoning_loop()`
   - Tier 2: `_call_gemini_final()`
4. Nếu Tier 2 lỗi quá ngưỡng retry -> fallback `_fallback_lite_as_flash()`.
5. Trả response đã được làm sạch artifact nội bộ.

Model aliases đang dùng:
- `REASONING_MODEL_ALIAS = "gemini-flash-lite-latest"`
- `FINAL_MODEL_ALIAS = "gemini-flash-latest"`
- `FALLBACK_MODEL_ALIAS = REASONING_MODEL_ALIAS`

Prompt loading:
- `src/core/prompt_loader.py` nạp prompt từ:
  - `src/instructions/azuris_system_prompt.txt`
  - `src/instructions/lite_reasoning_prompt.txt`
  - `src/instructions/fallback_system_prompt.txt`


================================================================================
2) GEMINI PIPELINE DETAILS
================================================================================

Tier 1 (reasoning + tool loop):
- Function: `_call_gemini_reasoning_loop()`
- Max retries: 5
- Max tool-loop iterations: 5
- Output: tuple `(reasoning_output, tool_results)`
- Budget guard: tối đa 1 lần `web_search` cho mỗi user turn trong reasoning loop

Tier 2 (final synthesis):
- Function: `_call_gemini_final()`
- Max retries: 5
- Dùng 3-block context injection:
  - BLOCK 1: user input (truncated)
  - BLOCK 2: reasoning output
  - BLOCK 3: tool results

Fallback tier (lite-as-final):
- Function: `_fallback_lite_as_flash()`
- Max retries: 3
- Trigger khi final tier không khả dụng sau retry

Output hygiene:
- Làm sạch các artifact nội bộ như `<THINKING>`, `<tool_code>`, `<tool_result>` trước khi gửi user.

4) TOOLS & SEARCH STACK (CURRENT)
================================================================================

Tool declarations trong `src/tools/tools.py`: 6 tools
1. `web_search`
2. `get_weather`
3. `calculate`
4. `save_note`
5. `retrieve_notes`
6. `image_recognition`

Search flow:
- Primary: DuckDuckGo (`run_search_apis`, stream count đọc từ `GOOGLE_SEARCH_STREAMS`, clamp 1..2)
- Fallback providers (khi cần và có key): SerpAPI, Tavily, Exa
- Fallback chỉ chạy khi forced hoặc khi primary không đủ context theo logic sufficiency check
- Có cache cho web search trong `ToolsManager`


================================================================================
5) RATE LIMITING & RELIABILITY
================================================================================

Per-user message limit:
- `RATE_LIMIT_THRESHOLD = 6`
- `RATE_LIMIT_WINDOW = 90` giây

Global request controls:
- `API_REQUEST_SEMAPHORE = asyncio.Semaphore(2)`
- `MIN_REQUEST_INTERVAL = 0.6` giây

Transient error handling (Gemini):
- Retry path bao gồm các nhóm lỗi: `429`, `resource_exhausted`, `unavailable`, `service unavailable`, `overloaded`
- Backoff dạng tăng dần theo attempt
- Trong reasoning tier, trạng thái reasoning/tool-results được giữ qua retry
